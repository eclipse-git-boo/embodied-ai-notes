## 2. 动机：为什么“有机器人数据”仍不够

大规模机器人数据昂贵且覆盖有限；人类视频和互联网图文包含更丰富的物体、语言和任务先验，却缺少与机器人动作一一对应的标注。π0.5 试图把这些信息接到可执行机器人策略上：高层视觉语言能力帮助识别任务、物体和常识，低层连续 action policy 负责满足具体机器人控制约束。

这不是把网页知识直接变成关节命令。论文的核心难题是**接口**：异构数据怎样进入同一个训练配方，同时避免语义训练淹没动作学习，或动作数据破坏已有 VLM 表征。

## 3. 方法：语义能力与动作流的协同

π0.5 延续 π0 的条件 flow matching：给定观测、任务条件和噪声动作块，预测从噪声向真实动作的向量场，并经少量采样步得到连续 action chunk。其开放世界部分来自多源训练与能利用语言/视觉上下文的策略设计，而不是单一的“搜索工具”。

<div class="method-flow"><span>图像 / 语言 / 任务上下文</span><b>→</b><span>VLM 语义表征</span><b>→</b><span>π0.5 action flow</span><b>→</b><span>短 chunk</span><b>→</b><span>闭环重观测</span></div>

对复现者，最可迁移的部分是：LeRobot/openpi 数据协议、动作/状态归一化、chunk/频率、训练与推理一致的 transform，以及从官方 checkpoint 做后训练。最不可迁移的是论文级别的异构预训练语料规模。

## 4. 实验应该怎样读

论文在真实世界任务与泛化设置中考察长时程执行、语义理解与新物体/场景适应。不要把展示视频当成统计结论：检查试验次数、初始位姿扰动、失败定义、是否允许人工重置，以及是否在同一相机/控制配置下比较。

对自己的复现，建立三级指标：① 离线 action / chunk 误差；② 固定台面、固定物体的闭环成功率；③ 变化光照、物体或起始位姿的泛化率。只有前两级稳定，才讨论“开放世界”。

## 5. 讨论与可复现性审计

| 问题 | 学习结论 |
|---|---|
| 论文真正贡献 | 不是个人可从头训练的模型大小，而是将 VLM 语义和连续策略置于统一实践路线。 |
| 公开资源 | openpi 提供最关键的训练/推理入口；论文完整预训练数据与配方不等于全部可得。 |
| 主要缺口 | 多源数据的精确比例、筛选细节、真实机器人接口与安全栈很难由论文完全复原。 |
| 当前最佳路线 | 官方 checkpoint → 自有单任务 LeRobot 数据 → 冻结/LoRA 小规模后训练 → 离线与仿真验证。 |

## 6. 算力、硬件与预期风险（学习规划，不在本机部署）

| 目标 | 建议配置 | 预期 |
|---|---|---|
| 数据检查 / 离线推理 | 24 GB GPU 起；NVMe 1–2 TB | 可先验证相机、时间戳、action schema。 |
| 小规模后训练 | 48 GB 单卡可作为起点，BF16 + gradient checkpoint / LoRA | 适合单机器人、单到少任务；需保留验证 episode。 |
| 全参后训练 | 推荐 4–8 × 80 GB 级 GPU | 视模型版本、序列长度和视觉输入而变；不是本阶段必需。 |
| 论文级预训练 | 大规模多机集群 + 海量异构数据 | 不应以此作为“复现成功”的判据。 |
| 真机接口 | 独立急停、限位/速度限制、相机固定、日志盘 | 首先保护人和设备；模型服务故障必须 fail-safe。 |

**最可能的失败原因**：动作坐标系或 norm stats 不一致、控制延迟超过训练分布、数据演示不含恢复行为、少量数据上的过拟合。将它们逐项记录到 [复现前自测](../../09-pi05复现前自测.html) 中。
