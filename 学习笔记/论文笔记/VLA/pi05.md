## 一句话结论

π₀.₅不是“给 π₀多喂一些机器人数据”，而是把低层连续控制、高层语义子任务和图文/规划数据放进同一 VLA 的协同训练框架，以争取在未见家庭环境中的长时程泛化。

## 模型与训练

- 主干：PaliGemma VLM + action expert；输入多视角图像、语言和 token 化 proprioception。
- 低层：flow matching 生成连续动作 chunk，适合快速控制。
- 高层：预测语义子任务，再以其调节低层动作，服务长时程任务分解。
- 数据：移动操作数据之外，还混合其它机器人、实验室任务、图文/视觉问答、规划和语言监督；论文首阶段绝大多数样本不是移动机械臂真实家庭数据。

## 与本项目真正相关的部分

开源 `openpi` 提供的是面向自有数据的微调基础设施：数据先转为 LeRobot 格式；然后以 Inputs/Outputs、DataConfig 与 TrainConfig 定义映射；最后由 policy server 提供推理。公开库可在 LIBERO 上验证 π₀.₅微调与推理。**这不等于论文中完整异构协同预训练可由单张 4090 重建。**

因此可验证目标应分层：

1. 在官方 LIBERO 配方上复现一次训练/评测闭环；
2. 把单一本体数据正确转为 LeRobot，并完成离线动作/观测映射检查；
3. 只在安全适配后做小任务微调和受限真机评测；
4. 不把“家庭长时程泛化”作为单机微调的预期。

## 最关键风险

| 风险 | 为什么会发生 | 防护 |
| --- | --- | --- |
| 动作定义错 | π₀.₅的训练/推理映射本体相关 | 写清坐标系、单位、delta/absolute，做离线 round-trip |
| 视觉错位 | 相机顺序、裁剪、时间戳不一致 | 固定数据契约，保存原始 episode 与可视化 |
| 归一化泄漏 | 用测试集或不匹配 stats | 仅由训练集生成 stats，并把版本写入 checkpoint |
| 过度承诺泛化 | 小数据微调覆盖很窄 | 固定 OOD 盲测、报告失败视频 |
| 控制风险 | 模型输出未经过约束 | 独立限幅、速度限制、急停；先离线、再空载 |
