# 环境基线（2026-07-15）

## 本工作机检测结果

| 项 | 检测值 | 结论 |
| --- | --- | --- |
| GPU | GeForce GTX 1660（6 GB，`nvidia-smi`） | **不是**目标中的 RTX 4090 48 GB |
| NVIDIA 驱动 | 457.49，CUDA 11.1 | 过旧，不适合当前 PyTorch / VLA 训练栈 |
| Python | 3.9.7 | 可用于部分工具，但建议为项目单独建环境 |

因此，不能把当前终端当作 4090 训练主机，也不应在此环境安装或验证 CUDA 版 LeRobot/OpenVLA。请在实际 4090 主机运行：

```powershell
nvidia-smi
python --version
```

并把输出填入实验记录。若 4090 是另一台机器，建议把本仓库放在 Git 仓库或同步盘，在 4090 主机克隆/同步后执行训练。

## 建议的软件边界

- **真机采集主机**：优先 Linux（Ubuntu 22.04/24.04）或受官方支持的 LeRobot 环境；需要稳定 USB 串口与相机访问。
- **训练主机**：4090 48 GB，使用与所选 PyTorch 对应的 NVIDIA 驱动；训练环境与真机采集环境可分开。
- **初始策略**：依照 LeRobot 官方 SO-101 页面安装其当前版本，使用 `so101_follower` 和 `so101_leader` 工作流；不要复制过期教程中的 `so100` 参数。
- **隔离**：ACT/SmolVLA、OpenVLA、openpi 分别创建环境，锁定依赖和 commit，避免 CUDA/PyTorch 冲突。

## 下一步的最小验收命令

在实际连接 SO-ARM101 的主机上，按 [官方 SO-101 文档](https://huggingface.co/docs/lerobot/en/so101) 完成安装与标定，再运行官方遥操作命令。仅在确认端口后替换占位符：

```bash
lerobot-teleoperate \\
  --robot.type=so101_follower --robot.port=<FOLLOWER_PORT> --robot.id=so101_follower \\
  --teleop.type=so101_leader --teleop.port=<LEADER_PORT> --teleop.id=so101_leader
```

先低速观察每个关节方向和急停，再开始 5 分钟稳定性测试。端口、标定文件和任何异常写入 [硬件验收表](01-硬件验收.md)。
