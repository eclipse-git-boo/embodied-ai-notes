# 学习模式说明

## 边界

本电脑只用于阅读、写笔记、浏览数据格式和设计未来实验。以下动作暂不执行：安装 CUDA/PyTorch/LeRobot、下载大模型权重、连接串口/相机、驱动 SO-ARM101 或 NERO、训练与真机部署。

保留命令的目的，是让你在将来使用训练/真机主机时能理解每一步的输入、输出与风险；它们不是当前待办事项。

## 推荐学习节奏

每篇论文分两次读：

1. **结构阅读（30–60 分钟）**：只回答任务、输入、输出、数据、训练目标、评测和结论。
2. **复现阅读（60–90 分钟）**：追代码/附录，画出张量形状、动作定义、控制频率和数据处理链；写出把它接到 SO-ARM101 时的未知量。

不要以“看完”为完成条件。每篇论文至少产出一页自己的笔记，使用 [论文阅读卡](模板-论文阅读卡.md)。

## 学习完成的证据

- 能脱离论文解释模型如何将 `图像 + 指令 + 机器人状态` 变成动作。
- 能说明一个数字（成功率或 benchmark 分数）对应的数据分布与评测条件。
- 能指出论文方案迁移到 SO-ARM101 前必须验证的动作空间、坐标系、相机和控制频率差异。
- 能为同一任务写出 ACT、VLA、WAM 三种方案的公平对照。
