# 从论文到工程的知识地图

论文里的成功率不是一个可直接迁移的属性。要把它翻译成自己的系统，需要经过以下接口。

```text
任务定义
  └─ 成功判据、物体、初始状态分布、语言集合
       ↓
硬件与观测
  └─ 机械臂、夹爪、相机、坐标系、时间同步、安全边界
       ↓
演示数据
  └─ 观测 / 动作 / 语言 / 时间戳 / 结果 / 视频 / 元数据
       ↓
数据契约与质检
  └─ 单位、维度、缺帧、异常动作、训练/测试划分
       ↓
策略或 WAM
  └─ 预训练表征 + 动作头；或 action-conditioned future prediction
       ↓
离线评估
  └─ loss、动作误差、预测误差（只用于诊断）
       ↓
闭环评测
  └─ 成功率、时间、碰撞、干预、延迟、失败视频
```

## 数据契约：未来采集时必须能回答

| 字段 | 要问的问题 | 常见错误 |
| --- | --- | --- |
| 图像 | 哪些相机？色彩格式、分辨率、FPS、时间戳？ | 图像帧和动作不是同一时刻 |
| 状态 | 关节角、速度、夹爪、末端 pose 的单位与坐标系？ | degree/radian 或左右手系混用 |
| 动作 | 是绝对目标、相对增量、关节命令还是速度？ | 将论文的动作含义当成自己的动作含义 |
| 语言 | 指令模板、同义表达、任务 ID 如何保存？ | 训练/测试语言完全重复却宣称语言泛化 |
| episode | 起止、成功标签、失败原因、操作者和场景版本？ | 只保存成功数据，未知失败模式 |

## 公平比较策略的最低要求

ACT、VLA 和 WAM 辅助策略比较时，至少固定：任务成功定义、测试初始状态、硬件标定、相机位置、最大执行时长、安全约束和人工干预规则。允许不同模型有不同输入时，应额外报告该差异，不能把“更多相机/更多状态”误写成算法收益。

## 4090 48 GB 在学习中的正确位置

显存影响未来可用的 batch size、图像分辨率、上下文长度和微调方法（LoRA/全参），但不改变上面的工程接口。阅读论文时请分别记录：模型参数量、精度、优化器状态、激活占用、数据吞吐和推理延迟，避免仅凭参数量判断是否可复现。

## 两套本体带来的学习价值

SO-ARM101 最适合学习低成本遥操作、LeRobot 数据格式和单臂桌面策略。NERO 的准确型号与接口尚未确认；未来将其视为一次“本体迁移”练习：哪些数据字段和策略可复用，哪些必须重新标定/重采集。不要把两个本体的数据混合后默认它们可交换。
