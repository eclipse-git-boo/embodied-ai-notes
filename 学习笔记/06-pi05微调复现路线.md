## 成功定义

本阶段不是重训论文的通用家庭机器人，而是在锁定版本的官方 `openpi` 路径上，完成 **LIBERO 官方配置 → 自有 LeRobot 格式数据的离线正确性 → 受限真机评测** 三层证据链。每层通过才进入下一层。

## 0. 先锁定可复现对象

- 锁定 `Physical-Intelligence/openpi` commit、Python/CUDA/PyTorch/JAX 或 PyTorch 后端、checkpoint 名称和数据集 revision。
- 阅读 [π₀.₅笔记](论文笔记/VLA/pi05.html) 与仓库 README；记录 README 中当时支持的后端与特性。不要把 GitHub issue、旧教程或不同后端的命令混用。
- 目前已知：openpi 提供 π₀/π₀-FAST/π₀.₅ 与 LIBERO 示例；其 PyTorch 路径支持情况与 JAX 路径并不完全相同，π₀-FAST、LoRA、mixed precision 等功能要以目标 commit 的说明为准。

## 1. 官方离线基线（未来训练主机执行）

目标：不改模型代码和数据语义，只跑通官方 π₀.₅ LIBERO 训练与评测说明。

1. 建独立环境，记录 `nvidia-smi`、驱动、依赖 lockfile 与仓库 commit。
2. 下载官方规定的 base checkpoint 与预转换 LIBERO 数据；校验文件大小/哈希。
3. 运行官方提供的 π₀.₅ LIBERO 配置，保存完整 stdout、config、seed、checkpoint 和评测结果。
4. 在同一 commit 上复跑一次小步数实验；确认曲线与 checkpoint 可恢复。

通过条件：训练损失、评测脚本、动作维数、观测键和 checkpoint 加载均无隐式修改。若失败，先定位版本/数据/权重，而不是调学习率。

## 2. 自有数据适配（未来训练主机执行）

openpi 的官方顺序是：**转 LeRobot 数据 → 定义 Inputs/Outputs 与 DataConfig → 定义 TrainConfig/权重加载 → 训练 → policy server 推理**。对 SO-ARM101/NERO，先写数据契约，再写代码。

### 数据契约最低字段

| 类别 | 必须确定 |
| --- | --- |
| 图像 | 每路相机的键名、RGB 顺序、尺寸、时间戳和相机顺序 |
| 状态 | 关节/夹爪/末端状态的单位、维数、参考系、采样时刻 |
| 动作 | 关节或末端、绝对或 delta、夹爪语义、频率、action horizon |
| 任务 | 语言 prompt、task ID、episode 成功/失败、场景版本 |
| 划分 | 按 episode 且按场景/物体切分，严禁相邻帧跨训练测试 |

### 三个离线闸门

1. **可视化闸门**：随机抽取 episode，视频、状态、动作曲线时间一致；动作逆归一化后与原始记录一致。
2. **统计闸门**：仅用训练集计算 norm stats；检查每个维度均值、方差、范围与物理限位合理。
3. **过拟合闸门**：在极小数据子集上可明显过拟合；不能过拟合通常是键名、掩码、动作或时间对齐错误。

## 3. 微调策略

先从单任务、固定相机、固定工作区、50–100 条高质量成功演示开始。保留 10 条验证和 20 条未参与训练的盲测初始状态。第一轮只改变一个变量：数据量、学习率、训练步数、图像增强或 action horizon 之一。

记录：数据 revision、config、seed、训练曲线、权重、离线可视化、每次真机视频。模型选择以盲测成功率和安全事件为准，不以训练 loss 最低为准。

## 4. 部署架构（未来真机机器执行）

```text
相机/状态采集 → 时间同步与预处理 → π₀.₅ policy server → 动作后处理/限幅 → 机器人控制器
        ↑                                                            │
        └────────────────── 真实下一帧闭环 ──────────────────────────┘
```

- policy server 与机器人驱动可分开运行，但必须记录网络与推理端到端延迟；
- 只能执行短 action chunk，并以新观测滚动重规划；
- 后处理层独立负责单位转换、关节/速度/工作空间限幅和急停，不能信任模型自行安全；
- 先做无机械臂推理、再空载低速、再单物体固定任务，最后才扩大场景。

## 风险登记

| 优先级 | 风险 | 早期信号 | 处置 |
| --- | --- | --- | --- |
| P0 | 动作空间或归一化错 | 机械臂方向反、幅度异常、仿真/回放不一致 | 停止真机；回到 round-trip 和 stats 检查 |
| P0 | 延迟导致闭环不稳 | 振荡、追赶过冲、动作滞后 | 缩短 chunk、降低速度、测量全链路延迟 |
| P0 | 硬件伤害 | 接近限位/碰撞/通信失联 | 独立急停、软限位、看护人员、低速空载 |
| P1 | 数据泄漏与伪泛化 | 测试视频与训练极相似 | episode/场景级切分，固定盲测集 |
| P1 | 过拟合 | 训练 loss 低、盲测失败 | 扩展初始状态/光照/物体覆盖，先修演示一致性 |
| P2 | 论文差距误读 | 小数据结果不及论文 | 区分预训练、异构协同训练和下游微调，不作不公平比较 |
